1. XML (eXtensible Markup Language) - это язык разметки, используемый для структурирования данных и обмена информацией между 
различными системами.
2. Первая строка в коде XML-документа, известная как объявление XML, содержит информацию о версии XML и о кодировке документа. 
Она обычно начинается со строки <?xml version="1.0" encoding="UTF-8"?>.
3. Правила синтаксиса XML-документа включают использование угловых скобок для обозначения тегов, разделение открывающего и 
закрывающего тегов слэшем, атрибуты должны быть заключены в кавычки и т.д.
4. Имена элементов в XML должны начинаться с буквы или символа подчеркивания и не могут содержать пробелы или специальные символы,
 кроме дефиса и точки.
5. Правило соблюдения корректной вложенности означает, что каждый открывающий тег должен иметь соответствующий ему закрывающий тег,
 и вложенные теги должны быть правильно вложены друг в друга.
6. Корневыми элементами в XML-документе являются элементы, которые являются непосредственными детьми документа и не имеют 
собственных родителей.
7. Валидация XML-документа позволяет проверить его на соответствие определенным правилам, заданным в схеме или DTD. Это помогает 
обнаружить ошибки и удостовериться, что документ структурирован корректно.
8. Простые типы элементов в XML Schema содержат только текстовую информацию, а комплексные типы могут содержать другие элементы и
 атрибуты.
9. Преимущество XML Schema перед DTD заключается в большей гибкости и возможности задать более сложные правила проверки и типы
 данных, а также в поддержке пространств имен.
10. Некоторые типы элементов XML Schema включаются, такие как string, integer, boolean, date и т.д.
11. xs:element - это элемент в XML Schema для определения элемента в XML-документе. Он указывает имя элемента, его тип данных и 
другие атрибуты.
12. xs:complexType - это элемент в XML Schema, который определяет составной тип элемента, который может содержать другие элементы
 и атрибуты.
13. xs:sequence - это элемент в XML Schema, позволяющий определить порядок элементов, которые могут содержаться в составном типе
 элемента.
14. xs:attribute - это элемент в XML Schema, используемый для определения атрибутов элемента. Атрибуты являются дополнительной 
информацией, которая может быть связана с элементом.
15. ENTITY в XML используется для определения и использования сущностей, которые могут быть использованы для замены определенных
 значений или фрагментов кода в XML-документе. Это может быть полезно, например, для определения специальных символов или ссылок 
 на внешние ресурсы.
16. Сущности в XML являются заменяемыми символами или строками, которые могут быть использованы в XML-документе. Они обычно 
используются для определения и переиспользования текстовых фрагментов или других данных, таких как ссылки на внешние файлы или др.

17. Встроенные сущности в XML включают:
- &amp; - амперсанд (&)
- &lt; - меньше (<)
- &gt; - больше (>)
- &quot; - двойная кавычка (")
- &apos; - апостроф (')

18. ATTLIST используется для определения атрибутов элемента в DTD. Он указывает имя элемента, а затем перечисляет атрибуты и 
их типы данных, значения по умолчанию, допустимые значения и другую информацию об атрибуте.

19. Валидный XML-документ - это документ, который соответствует определенным правилам (DTD или XML Schema). Типы валидного 
XML-документа могут быть "Well-Formed" (хорошо сформированы, соответствуют основным правилам синтаксиса XML) или "Valid"
 (правильно структурированы и соответствуют определенным правилам DTD или XML Schema).

20. Запись <!ENTITY name “Hello, world!”> означает, что в XML-документе сущность name определена и заменяется на строку 
"Hello, world!".

21. Запись <!ELEMENT to (#PCDATA)> означает, что элемент to может содержать только текстовую информацию (#PCDATA).

22. Инструкция ATTLIST содержит параметры:
- Имя элемента (имя элемента, для которого определяются атрибуты)
- Имя атрибута (имя атрибута, который будет определен для элемента)
- Тип данных (тип данных, который может принимать значение атрибута)
- Значение по умолчанию (значение, которое будет использоваться, если атрибут не указан)
- Допустимые значения (список допустимых значений, которые может принимать атрибут)
- Другие параметры (например, флаги, указывающие на обязательность атрибута, его уникальность и т.д.)

23. Инструкция ELEMENT имеет следующие параметры:
- Имя элемента (имя элемента, который будет определен в DTD)
- Тип данных (тип данных, который может содержать элемент или его содержимое, например, PCDATA, CDATA, EMPTY и др.)
- Другие параметры (например, параметры, указывающие на количество раз, которое элемент может встречаться в документе,
 ограничения на его содержимое и т.д.)

24. По адресу http://www.w3.org/2001/XMLSchema находится спецификация XML Schema, языка для определения структуры, типов
 данных и ограничений для XML-документов.

25. DTD можно объявить внутри XML-документа с помощью <!DOCTYPE> инструкции. Аббревиатура DTD расшифровывается как Document
 Type Definition (Определение типа документа).
